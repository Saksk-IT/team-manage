第一章 准备工作

目前 GPT Team 已全部失效，后续可以接入中转使用 GPT Codex。中转使用纯血 Plus 号池，支持 gpt-5.5，并支持 Image 2.0 生图。

Codex.app 是 GPT 官方推出的强大软件，具备网页版的核心能力，并且在本地客户端中提供更强的使用体验。

兑换码来源都适用。 本教程同时适用于按邮箱质保剩余时间补发的中转兑换码，也就是质保补发的中转兑换码，以及在链动小铺购买的额度包兑换码。
额度兑换码：必须选择 GPT-Plus 链动小铺购买的额度包 / 余额包兑换码，新建 API Key 时必须选择“GPT-Plus”。不能选“质保补偿”，否则无法使用。
质保补发码：选择质保补偿 通过质保网站补发的中转兑换码，在新建 API Key 时选择“质保补偿”分组。
订阅模式 每天有一定额度，每天 0 点后自动刷新。
额度模式 使用时消耗网站账户里的额度，用完就没有了。

第二章 创建 API 密钥（所有接入方式通用）

重要｜本章是所有接入方式的前置步骤。无论后面使用手动文件配置，还是使用 cc-switch，都要先拿到自己的 API Key。

2.1 创建新密钥并选择正确分组

在“API 密钥”页面，点击“创建密钥”。名称可按自己需要随便填写，然后按兑换码来源选择分组。

订阅类分组：质保补偿 / 30刀订阅 “质保补偿”和“30刀订阅”是订阅制度，不需要消耗余额；质保网站补发的中转码选“质保补偿”。
额度类分组：GPT-Plus 链动小铺购买余额包或额度兑换码时，应选择“GPT-Plus”分组。创建密钥前请再次确认自己使用的是订阅还是额度。
分组选错会导致无法使用。 分组决定密钥使用的权益来源，请先判断兑换码来源，再选择对应分组。

点击“使用密钥”后会看到后续操作需要的配置内容。最终复制你自己的 config.toml 和 auth.json 内容；截图中的 API Key 已脱敏。

第三章 接入 Codex

3.1 下载 Codex 软件

下载好后必须先打开一次 Codex。 先打开 Codex 初始化配置文件，然后再完全关闭 Codex，继续下一步文件配置。
https://openai.com/zh-Hans-CN/codex/

3.2 配置方式一：手动文件配置

配置文件前必须完全关闭 Codex。 配置文件时一定要保持 Codex 进程关闭。请确保 Codex 进程没有在运行，再编辑 config.toml 和 auth.json，否则配置可能不会生效。
系统 配置目录 打开方式
Windows %userprofile%\.codex 按 Win + R，输入目录并回车；文件双击没有反应时，右键选择“记事本”打开。
Mac ~/.codex 可运行 mkdir -p ~/.codex 创建目录；访达中看不到隐藏文件时按 Option + Shift + .。

提示：如果目录或文件不存在，可手动新建 .codex 目录，以及 config.toml / auth.json 两个文件。

后续操作 Mac 和 Windows 一样：把你自己获得的内容分别粘贴到 config.toml 和 auth.json 文件中。不要粘贴截图里的脱敏内容。

3.3 配置方式二：使用 cc-switch 一键配置

新手可以使用 cc-switch 一键配置秘钥。下载后选择右上角“+”，选择 Codex 供应商，再选择自定义配置。

cc-switch releases
  1. API KEY 填写上一步创建得到的 API Key。
  2. API 地址填写 https://api.sakms.top/。
  3. 回到主页面点击开启，重新打开 Codex 即可正常使用。

3.4 重新打开 Codex 并使用 API 登录

完成文件配置后重新打开 Codex，选择“换种方式登录”，再选择 API 登录并粘贴自己的 API 密钥。

第四章 验证与排错

4.1 登录失败时快速检查

  • 编辑配置前是否已经完全关闭 Codex？
  • config.toml 和 auth.json 是否放在同一个 .codex 目录？
  • 是否粘贴了你自己生成的 API Key，而不是教程截图中的示例？
  • 创建密钥时是否按来源选择正确分组：质保网站补发的中转码选“质保补偿”，链动小铺额度兑换码选“GPT-Plus”？
  • 如果使用 cc-switch，API 地址是否填写为 https://api.sakms.top/，并在主页面点击开启后重新打开 Codex？

4.2 常见问题

现象 原因 处理方式
401 / Incorrect API key 配置时 Codex 已经在运行，或 API Key 粘贴错误。 关闭 Codex 软件，重新配置 config.toml，再用 API 密钥登录。
原聊天记录不见 切换 API 中转后，本地 provider 不一致。 参考 codex-provider-sync releases 恢复。
429 Too Many Requests 一般是当前额度或频率已经触发限制。 打开额度查询页面 查看余额和额度。
