03

创建并查看 API 密钥

在“API 密钥”页面新建密钥，按兑换码来源选择对应分组，然后点击“使用密钥”获取后续配置内容。

分组选择规则。 “质保补偿”和“30刀订阅”是订阅制度，不需要消耗余额；在“链动小铺”购买余额包的，应选择“GPT-Plus”分组。

创建密钥

质保补发的中转码选“质保补偿”或“30刀订阅”；链动小铺余额包选“GPT-Plus”。

3.3

使用密钥

05

配置方式一：手动配置 Codex 文件

找到或创建 .codex 配置目录，然后将“使用密钥”页面给出的内容分别写入 config.toml 和 auth.json。

Windows 用户
  1. 按 Win + R 打开运行窗口。
  2. 输入 %userprofile%\.codex 并回车。
  3. 如目录不存在，请手动创建 .codex 文件夹。
  4. 在目录中找到或新建 config.toml 和 auth.json。
  5. 双击文件没有反应时，可右键选择“记事本”打开。
Mac 用户
  1. 可运行 mkdir -p ~/.codex 创建配置目录。
  2. 在访达用户目录中找到 .codex 文件夹。
  3. 找不到隐藏文件时，按 Option + Shift + . 显示隐藏文件。
  4. 在目录中找到或新建 config.toml 和 auth.json。
粘贴位置要对应。 把“使用密钥”页面显示的 config.toml 内容粘贴到 config.toml，把 auth.json 内容粘贴到 auth.json。不要复制教程截图中的脱敏内容。
06

配置方式二：使用 cc-switch 一键配置

新手也可以下载 cc-switch，通过图形界面添加 Codex 供应商，填入 API Key 和 API 地址后启用。

cc-switch releases
1. 下载并打开 下载 cc-switch，打开后点击右上角“+”。
2. 添加供应商 选择“Codex 供应商”，再选择“自定义配置”。
3. 填写配置 API KEY 填写上一步创建得到的 API Key，API 地址填写 https://api.sakms.top/。
4. 开启使用 回到 cc-switch 主页面点击开启，重新打开 Codex 即可正常使用。
07

重新打开 Codex 并使用 API 登录

完成文件配置后重新打开 Codex，选择“换种方式登录”，再选择 API 登录并粘贴自己的 API 密钥。

检查

登录失败时快速检查

如果无法登录或 Codex 没有读取到配置，请按下面顺序排查。

  • 编辑配置前是否已经完全关闭 Codex？
  • config.toml 和 auth.json 是否放在同一个 .codex 目录？
  • 是否粘贴了你自己生成的 API Key，而不是教程截图中的示例？
  • 创建密钥时是否按来源选择正确分组：质保补发的中转码选“质保补偿”或“30刀订阅”，链动小铺余额包选“GPT-Plus”？
  • 如果使用 cc-switch，API 地址是否填写为 https://api.sakms.top/，并在主页面点击开启后重新打开 Codex？
FAQ

常见问题

按教程操作后仍异常时，优先检查 Codex 是否读取了最新配置，以及原聊天记录是否需要单独同步。

问题一：登录时报 401 / Incorrect API key

通常是因为配置 config.toml 时 Codex 已经在运行，客户端没有读取到新的中转配置。

  1. 关闭 Codex 软件。
  2. 重新配置 config.toml。
  3. 重新使用 API 密钥登录。
问题二：原 Codex 聊天记录恢复

使用 API 中转后，部分用户可能看不到原来的聊天记录。可参考下载同步工具恢复。

codex-provider-sync releases
问题三：提示 429 Too Many Requests / 限额

如果出现 exceeded retry limit、429 Too Many Requests 等提示，一般是当前额度或频率已经触发限制。
