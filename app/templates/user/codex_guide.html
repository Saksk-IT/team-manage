说明

使用前请先确认

兑换码来自中转服务，并已按质保剩余时间更新。页面中的密钥截图已脱敏，请务必使用你自己页面中生成的 API Key。

配置文件前必须完全关闭 Codex。 请确认 Codex 进程没有运行，再编辑 config.toml 和 auth.json，否则配置可能不会生效。
账号与密钥安全。 注册中转账户时不要复用重要账户密码；API Key 只粘贴到自己的 Codex 配置或登录框中，不要发给他人。
01

打开中转链接并创建账户

访问注册页，随意输入邮箱和密码即可创建中转账户。

https://api.sakms.top/register
02

在兑换页面输入兑换码

登录后进入兑换页面，输入提供给你的补偿兑换码并点击“兑换”。

03

创建并查看 API 密钥

在“API 密钥”页面新建密钥，按兑换码来源选择对应分组，然后点击“使用密钥”获取后续配置内容。

3.1 创建密钥 点击“创建密钥”。
3.2 选择分组 质保补偿码选“质保补偿”；链动小铺余额制选“GPT-Plus”。
3.3 使用密钥 进入使用密钥弹窗，复制自己的配置内容。
分组选择规则。 “质保补偿”是订阅制度，不需要消耗余额；在“链动小铺”购买的是余额制，应选择“GPT-Plus”分组。
04

下载 Codex 软件

前往 OpenAI Codex 下载页，按你的系统下载并安装客户端。

https://openai.com/zh-Hans-CN/codex/
05

配置 Codex 文件

找到或创建 .codex 配置目录，然后将“使用密钥”页面给出的内容分别写入 config.toml 和 auth.json。

Windows 用户
  1. 按 Win + R 打开运行窗口。
  2. 输入 %userprofile%\.codex 并回车。
  3. 如目录不存在，请手动创建 .codex 文件夹。
  4. 在目录中找到或新建 config.toml 和 auth.json。
Mac 用户
  1. 可运行 mkdir -p ~/.codex 创建配置目录。
  2. 在访达用户目录中找到 .codex 文件夹。
  3. 找不到隐藏文件时，按 Option + Shift + . 显示隐藏文件。
  4. 在目录中找到或新建 config.toml 和 auth.json。
粘贴位置要对应。 把“使用密钥”页面显示的 config.toml 内容粘贴到 config.toml，把 auth.json 内容粘贴到 auth.json。不要复制教程截图中的脱敏内容。
06

重新打开 Codex 并使用 API 登录

完成文件配置后重新打开 Codex，选择“换种方式登录”，再选择 API 登录并粘贴自己的 API 密钥。

检查

登录失败时快速检查

如果无法登录或 Codex 没有读取到配置，请按下面顺序排查。

  • 编辑配置前是否已经完全关闭 Codex？
  • config.toml 和 auth.json 是否放在同一个 .codex 目录？
  • 是否粘贴了你自己生成的 API Key，而不是教程截图中的示例？
  • 创建密钥时是否按来源选择正确分组：质保补偿码选“质保补偿”，链动小铺余额制选“GPT-Plus”？
FAQ

常见问题

按教程操作后仍异常时，优先检查 Codex 是否读取了最新配置，以及原聊天记录是否需要单独同步。

问题一：登录时报 401 / Incorrect API key

通常是因为配置 config.toml 时 Codex 已经在运行，客户端没有读取到新的中转配置。

  1. 关闭 Codex 软件。
  2. 重新配置 config.toml。
  3. 重新使用 API 密钥登录。
问题二：原 Codex 聊天记录恢复

使用 API 中转后，部分用户可能看不到原来的聊天记录。可参考下载同步工具恢复。
