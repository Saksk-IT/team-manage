Claude Code 完整接入流程

先准备好两项信息：在中转站创建自己的 API Key，并以“使用密钥”弹窗里 Claude Code 对应配置为准。教程截图和示例里的密钥都不能直接复制。

1. 从第一步开始：注册、兑换、创建 Key

  1. 打开 中转注册页，填写邮箱、验证码和密码，完成中转账户注册。
  2. 如果还没有权益，先通过卡密自助购买地址 https://pay.ldxp.cn/shop/LSSZLMUY 购买额度包，或使用已发放的质保补发兑换码。
  3. 登录后进入兑换页面，输入中转兑换码或额度包兑换码并兑换；随后打开 额度查询页 确认权益到账。
  4. 进入 API 密钥页面，点击“创建密钥”，按来源选择正确分组：质保补发码选“质保补偿”，链动小铺额度包选 GPT / GPT-Plus。
  5. 创建成功后点击“使用密钥”，切到 Claude Code 配置区域，复制弹窗里的真实 base_url 和 api_key。

2. 推荐方式：cc-switch 一键导入 Claude Code

新手优先使用 cc-switch。它会把中转配置写入本机 Claude Code 使用的环境，减少手动复制变量时写错 Key 或地址的概率。

下载 cc-switch releases
  1. 下载并打开 cc-switch，保持软件在运行状态。
  2. 打开 API 密钥页面，在自己的密钥操作区域点击“导入到 CCS”。
  3. 浏览器如弹出是否打开外部应用，请允许打开 cc-switch。
  4. cc-switch 中出现 Sak AI 后，点击“启用”。
  5. 关闭旧终端并重新打开新终端，再输入 claude 启动 Claude Code。

注意：一键导入后一定要重启 Claude Code，命令行用户还要关闭旧终端再打开新终端，否则旧环境变量可能还没有生效。

3. 备用方式：手动配置 Claude Code

如果 cc-switch 无法打开或需要在服务器上配置，可手动写入环境变量。Claude Code 官方支持在 settings.json 里通过 env 为每次会话注入变量；也可以直接写到系统环境变量中。

3.1 定位 Claude 配置目录

系统配置目录打开方式
Windows%userprofile%\.claude按 Win + R，输入 %userprofile%\.claude 并回车；目录不存在时可手动新建。
macOS~/.claude终端执行 mkdir -p ~/.claude && open ~/.claude。
Linux~/.claude终端执行 mkdir -p ~/.claude && cd ~/.claude。

3.2 方式 A：写入 settings.json（推荐备用）

在 ~/.claude/settings.json 中写入下面结构。ANTHROPIC_BASE_URL 和 ANTHROPIC_AUTH_TOKEN 请复制“使用密钥”弹窗里的真实值；如果弹窗给出的地址带 /v1，就照弹窗填写。

{
  "env": {
    "ANTHROPIC_BASE_URL": "https://api.sakms.top",
    "ANTHROPIC_AUTH_TOKEN": "填写你的 API 密钥",
    "ANTHROPIC_MODEL": "gpt-5.5"
  }
}

提示：如果文件里已经有其他设置，只新增或合并 env 字段，不要覆盖原有 permissions、hooks 等配置。

3.3 方式 B：配置系统环境变量

系统设置方法
Windows PowerShellsetx ANTHROPIC_BASE_URL "https://api.sakms.top"
setx ANTHROPIC_AUTH_TOKEN "填写你的 API 密钥"
macOS / zsh在 ~/.zshrc 末尾追加 export ANTHROPIC_BASE_URL="https://api.sakms.top" 和 export ANTHROPIC_AUTH_TOKEN="填写你的 API 密钥"，保存后执行 source ~/.zshrc。
Linux在 ~/.bashrc 或 ~/.zshrc 追加同样的 export 语句，再重新打开终端。

4. 验证与排错

  • 打开新终端窗口，输入 claude，能进入交互并发起一次对话即配置成功。
  • 如果提示认证失败，回到中转站重新复制 Claude Code 配置，并确认没有复制教程截图里的脱敏密钥。
  • 如果 cc-switch 已启用但仍无效，先退出 Claude Code，再关闭旧终端，重新打开终端后再次输入 claude。
  • 如果提示额度或限流，打开 额度查询页 检查余额、订阅日额度和分组是否正确。
