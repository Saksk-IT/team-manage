Claude Code 系列（CLI / VSCode 插件）

开始前请先完成：回到 Codex 总教程第二章 创建 API Key，并在“使用密钥”弹窗中复制 Claude Code 对应配置。不要复制教程截图中的示例密钥。

适用范围：Claude Code CLI、VSCode Claude 插件。配置方式有两种：环境变量或 settings.json，二选一即可。建议优先使用 settings.json，便于后续排查。

1. 定位 Claude 配置目录

系统配置目录打开方式
Windows%userprofile%\.claude按 Win + R，输入 %userprofile%\.claude 并回车；目录不存在时可手动新建。
macOS~/.claude终端执行 open ~/.claude；如不存在先执行 mkdir -p ~/.claude。
Linux~/.claude终端执行 cd ~/.claude 或直接进入；如不存在先执行 mkdir -p ~/.claude。

2. 方式 A：配置环境变量

复制“使用密钥”弹窗中 Claude Code 给出的环境变量，通常包含 ANTHROPIC_BASE_URL、ANTHROPIC_AUTH_TOKEN 等。下面示例仅用于说明，请把密钥替换成自己的真实 API Key。

系统设置方法
Windows控制面板 → 系统 → 高级系统设置 → 环境变量 → 新建用户变量；或用 PowerShell：setx ANTHROPIC_BASE_URL "https://api.sakms.top/v1"。
macOS在 ~/.zshrc 末尾追加：export ANTHROPIC_BASE_URL="https://api.sakms.top/v1"，保存后执行 source ~/.zshrc。
Linux在 ~/.bashrc 或 ~/.zshrc 末尾追加同上 export 语句，再执行 source。

3. 方式 B：修改 settings.json（建议）

进入上表对应的 .claude 目录，编辑或新建 settings.json，按“使用密钥”弹窗给出的内容填写。完成顺序：先设环境变量或改 settings.json → 重启终端 → 输入 claude 启动。

提示：旧终端不会自动读取新环境变量。修改完成后请关闭旧终端，打开新终端再启动 Claude Code。

4. 验证 Claude Code

  • 打开新终端窗口，输入 claude。
  • 能正常进入交互并发起一次对话，即表示配置成功。
  • 如果提示密钥错误，回到中转站重新复制 Claude Code 配置，并确认 API Key 没有泄露或被删除。
